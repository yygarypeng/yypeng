Projects

HWW quantum entanglement and W polarization

Problem: $H \to WW^* \to \ell\nu\ell\nu$ carries quantum-entanglement, spin-correlation, and W-polarization information, but two neutrinos hide key rest-frame variables.

Role and evidence: Kickoff member of the HWW quantum-entanglement / W-polarization project, working on missing-information reconstruction for MSc thesis and ATLAS internal-note development.

Relevance: quantum-information, diboson polarization, Higgs/EW precision, and missing-information final states where reconstruction must be interpretable.

ML reconstruction for missing information

Problem: Two unobserved neutrinos make standard kinematic reconstruction under-constrained, while the analysis still needs rest-frame variables.

Role: Developed validation-oriented ML reconstruction workflows; one model has been integrated into the central QE analysis framework.

Validation: Observable shapes, bias/resolution, truth-to-reco correlation, detector robustness, systematics behavior, and baseline comparison where public information allows.

Tools and relevance: PyTorch/Lightning, FrEIA, ONNX, ROOT/uproot/mplhep; relevant to neutrino reconstruction, flow/generative ambiguity sampling, and collaboration-review-ready ML.

Run 3 HWW coupling analysis

Problem: Run 3 measurements need continuity, portable environments, and contributor onboarding.

Role: First-look Run 3 HWW coupling-analysis studies, recent focus on control regions, workflow development, validation plotting, and MSc student mentoring.

Tools and relevance: FastFrames, ROOT, Python, reproducible environments, and real ATLAS analysis operations beyond isolated ML prototyping.

HGTD production database

Problem: Detector production needs reliable data management, quality-control views, and interfaces usable by multiple teams.

Role: Recent ATLAS qualification task: frontend/backend components for organizing, querying, and visualizing HGTD production data.

Tools and relevance: Django, Vue.js, database-backed interfaces; evidence of ATLAS service and detector-software contribution beyond physics analysis.

NTHU workstation and CernVM-FS

Problem: ATLAS analysis depends on reproducible software environments, reliable local computing, and access to collaboration software stacks.

Role: Main working facility for the NTHU experimental HEP group, supporting ATLAS analysis, ML training, containerized workflows, GPU resources, and CernVM-FS caching.

Tools and relevance: Docker/Podman, CernVM-FS, UNIX, ROOT, Python; useful for reproducible ML training/inference and reviewable ATLAS workflows.
